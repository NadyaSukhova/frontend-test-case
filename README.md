# Список исправленных при рефакторинге ошибок

## Ошибки в файле Redux с состоянием store.js

1. Единый слайс под весь проект - антипаттерн. 
Решение - разделение на разные слайсы.

2. Разные методы выполняют одну и ту же задачу, повторение данных (cartTotalAmount, cartTotalSum, totalPrice хранят одно и то же). То, что можно посчитать, хранится в состоянии (пр.: cartTotalAmount) - антипаттерн избыточных вычислений. 
Решение - пересмотр нужности каждого состояния.

3. Повторение кода. При любом изменении будет пересчет ВСЕХ полей - антипаттерн избыточных вычислений. 
Решение: перенести все ерерасчёты в селекторы.

4. При удалении товаров с помощью кнопки "-" (минуса) - при нулевой quantity товар все равно остается в корзине. 
Решение: добавить в экшн addToCart проверку и удалять товар в случае количества равным 0.

5. Обнаружено неиспользуемое состояние cartItemsCount, оно пересчитывается вручную, состояние не используется - антипаттерн избыточных вычислений.
Решение: удалить состояние.


## Ошибки в файле App.jsx

1. Использование селекторов и экшенов не из слайсов. 
Решение: импортировать селекторы и экшены из слайсов.

2. Один файл на всё приложение. Это неправильный подход к разработке 
Решение: создание отдельной папки под компоненты, разбивание компонентов по разным файлам.

## Инструкции по запуску не поменялись

1. Установите зависимости:
```bash
npm install
```

2. Запустите проект:
```bash
npm run dev
```

3. Откройте браузер по адресу `http://localhost:5173`
